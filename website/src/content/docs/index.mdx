## Why TRIGRA?

<CardGrid stagger>
  <Card title="Universal Applier" icon="rocket">
    Standard Kubernetes orchestration for any resource type. Deploy CRDs, Helm releases, or raw YAML with zero friction.
  </Card>
  <Card title="Edge & Homelab Optimized" icon="laptop">
    Designed for resource-constrained environments. Perfect for multi-node K3s/RKE2 clusters and edge computing.
  </Card>
  <Card title="Instant Auto-Detection" icon="setting">
    Zero-config intelligence. TRIGRA detects if it's running in-cluster or remotely and configures its context accordingly.
  </Card>
  <Card title="Enterprise Security" icon="approve-check">
    Production-grade security with SHA-256 webhook validation, scoped RBAC, and native Kubernetes secret integration.
  </Card>
  <Card title="Standardized Workflow" icon="pencil">
    Eliminate 'kubectl' manual apply. Commit to Git, and let the controller handle atomic updates and state reconciliation.
  </Card>
  <Card title="Modern Tooling" icon="document">
    Built for the modern stack. Includes first-class Helm support, CI/CD integration, and a sleek developer portal.
  </Card>
</CardGrid>
